Нужно написать парсер ВБ, который на входе получает список артикулов ВБ с сопоставленными ему поисковыми запросами из CVS файла.

При запуске парсера задается значение периодичности парсинга в часах.

Парсер будет запускаться на винде.

На выходе формирует новый CVS файл формата UTF-8 в котором по столбцам записаны данные, спарсеные на ВБ:

articles_list
brand_list
sale_size_list
price_with_sale_list
price_spp_list
size_spp_list
name_list
url_list
have_list
rating_list
count_featback_list
count_buy_list
Артикул +
Бренд +
Размер скидки +
Цена со скидкой +
Цена с СПП +
Размер СПП +
Место в выдаче. Если прописан артикул, но нет запроса, данные по артикулу все равно парсятся, кроме места в выдаче, в этом случае данная ячейка не заполняется ( не 0 указывает, а оставляем пустой ячейку)
Название товара +
URL страницы товара +
Наличие в данные момент (1 если есть, 0 если нет) +
Рейтинг товара в числовом значении ( 4,3, 5,0 и т.д.)+
Количество отзывов +
Счетчик покупок, который указан в карточке. числовое значение из строки "купили более 200 раз" и т.д. в данном случае спарсеное значение будет 200 +
После каждого парсинга файл результатов перезаписывается.

Пример исходного файла articles, файл результата result, сам парсер тоже во вложении, можно посмотреть его работу. для этого в его папку нужно загрузить исходный файл, да парсер есть, но его нужно расширить и периодически появляются новые задания по его изменению, плюс другие задачи, поэтому и нужен человек на постоянное сотрудничество. Приоритет тем с кем можно рассчитываться официально
